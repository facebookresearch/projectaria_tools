## Getting Access to Support Channels

If you have received ARK access but haven't received invitations to Discord or Workplace:

1. **Check your email** (including spam/junk folders) for invitation links
2. **Verify your email address** matches the one used for ARK registration

## Before Requesting Support

To help us assist you more efficiently, please try these steps first:

### 1. Check the Documentation

Many common questions are answered in our documentation:

- **[Device Manual](/ark/device/manual)** - Hardware controls, indicators, and operation
- **[Client SDK Troubleshooting](/ark/support/sdk)** - Common SDK issues and solutions
- **[MPS Troubleshooting](/ark/support/mps)** - MPS processing issues
- **[Technical Specifications](/technical-specs)** - Detailed hardware and software specs

### 2. Review Known Issues

Check for known issues and workarounds:

- [GitHub Issues](https://github.com/facebookresearch/projectaria_tools/issues) - Open issues and bug reports
- Discord/Workplace announcements - Recent updates and known problems

### 3. Verify Your Setup

Common setup issues to check:

**Device Issues:**
- ‚úÖ Device is fully charged
- ‚úÖ Privacy switch is in correct position
- ‚úÖ Device firmware is up to date
- ‚úÖ Companion App is updated to latest version

**SDK Issues:**
- ‚úÖ Using supported Python version (3.9-3.12)
- ‚úÖ Client SDK is installed correctly
- ‚úÖ Device authentication is completed (`aria-gen2 auth check`)
- ‚úÖ Device is connected via USB or WiFi

**Data Issues:**
- ‚úÖ VRS file is not corrupted
- ‚úÖ Using compatible version of Project Aria Tools
- ‚úÖ Sufficient storage space available

## When Requesting Support

To help us resolve your issue quickly, please provide:

### Essential Information
1. **[Rage shake](/ark/support/rageshake) when issue just happens**

2. **Clear description of the issue**
   - What you were trying to do
   - What actually happened
   - Error messages (exact text or screenshots)

3. **Environment details**
   - Operating system and version
   - Python version
   - Client SDK version (`pip show projectaria-client-sdk`)
   - Project Aria Tools version (`pip show projectaria-tools`)

4. **Steps to reproduce**
   - Detailed steps to reproduce the issue
   - Whether it happens consistently or intermittently

5. **Device information** (if applicable)
   - Device serial number
   - Firmware version
   - Recording profile used
   - Battery level during issue

## Emergency Support

For critical issues that are blocking your research:

1. **Email** [AriaOps@meta.com](mailto:AriaOps@meta.com) with "URGENT" in subject line
2. Clearly explain the **impact** and **timeline** constraints
3. Provide all relevant details from the "When Requesting Support" section above

We'll prioritize urgent requests affecting:
- Active research studies with participants
- Conference or publication deadlines
- Large-scale data collection campaigns
- Critical demonstrations or presentations
